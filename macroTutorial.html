Templates

We can see templates as procedures that generate code. Usually, it also takes code as parameters. The input code is always given as an untyped parameter. Templates are designed to be an efficient copy-paste mechanism, very like #define macros in C language family (C++, C# as well). We can add an (already existing in fact) boolean operator, to simplify expressions. Notice we use stropping to overwrite the already existing operator.

## Example from std/manual
template `!=` (a, b: untyped): untyped =
  not (a == b)

doAssert(4 != 5)

We can easily repeat code under a custom block. Here duplicate, just duplicate code and repeat takes an additional parameter, an int, as a generalisation of the duplicate template.

template duplicate(statements: untyped) =
  statements
  statements

duplicate:
  echo 5

In standard Nim, there are very few restricted keywords and special control-flow mechanisms, but nothing restrains us from creating a doWhile construct. And this is very simple.

template doWhile(conditional, loop: untyped) =
  loop
  while conditional:
    loop

var myInt = 5
doWhile myInt >= 6:
    echo myInt
    if myInt mod 2 == 1:
      myInt *= 3
      myInt.inc
    else:
      myInt = myInt div 2

Macros

Macros can be seen as an empowered template procedure. Instead of simply taking untyped blocks and reuse them as lego bricks to return a code, we can parse our untyped parameter and do something conditionnaly to informations given in these parameters.
